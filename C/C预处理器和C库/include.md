```
该指令实际上是将被包含的文件直接复制到文件中

<> 与 "" 之间有区别
在UNIX(现在的大多数IDE也是)中，<>表示在标准系统目录中查找该文件，""则表示优先在当前目录或指定目录中查找文件
```

```
现在写代码推崇使用分离式让结构更清晰
在C中，头文件并不包含函数实现，头文件通常只包括define常量和预处理器指令(内联函数除外)

要让头文件连接源文件，只需要在源文件中用#include包含头文件即可，之后就可以通过头文件使得与源文件在同一编译单元中的代码使用头文件中定义、源文件中实现的函数了
```

```
在大多数的高级语言中标识名不能重复，在C中亦是如此，因此多次包含同一个文件成了头文件必须要防范的事，对此，有多种方法

一、用ifndef实现一次包含
#ifndef(_STDIO_H)
    #include <stdio.h>
#endif
通过多个上述语句实现一次包含

二、一的plus
将一的内容全部写在一个头文件中，通过包含该头文件实现一次包含

三、使用#pragma once
	该指令一般由编译器提供保证：一个文件不会被多次包含，这个一个文件指物理文件，如果是内容相同的文件则无法识别
	而且由于该指令是设置编译器的，所以移植性差。
	如果你的头文件有多份备份，那么该指令也是无法识别的，还是会导致文件重复包含的问题
	同时，使用第一种方法由于需要每个文件都确认，所以三会稍微快一点
```
